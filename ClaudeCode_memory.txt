4. TOPOGRAPHY & DATUM CORRECTION (rule 03a)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Active source: config.yml → topography.source = "DiluviumDEM"
Clip elevation: topography.clip_elevation_m.DiluviumDEM = 30 m

Pipeline in 03a_get_elevation.py:
  1.  Merge DiluviumDEM tiles → topo_utm (UTM, NaN ≥ clip_elevation_m)
  1b. [DiluviumDEM + datum_correction.enabled] Vertical datum correction:
        Step 1: topo_utm += geoid_offset  (N_EGM2008 − N_GOCO06s, pyshtools)
        Step 2: topo_utm -= MDT_extrapolated_over_land
      KEY DESIGN: MDT (NaN over land) must be extrapolated over land using IDW
      (rasterio.fill.fillnodata) BEFORE resampling to DEM grid. Without this,
      only coastal fringe pixels get MDT correction; inland pixels only get the
      geoid shift, creating a discontinuity.
  2.  Load GEBCO bathymetry → gebco_utm
  3.  Land mask from OSM polygons
  4.  Fill land NaN → land_barrier_val (impassable barrier = 1e7 m)
  5.  Hard-boundary merge: DEM on land, GEBCO offshore → elevation_merged.tif
  6.  Optional gradient blend (blend.enabled: false by default) → side-effect only
  7.  zsini.tif (0 over ocean, DEM value on land, nodata elsewhere)

Datum correction config (topography.datum_correction):
  enabled: true
  mdt_variable: "mdt"
  mdt_load_margin_deg: 5.0   ← wide margin ensures sufficient offshore MDT pixels

Debugging plots produced as side-effects when datum_correction.enabled:
  01b_geoid_offset.png     — N_EGM2008 − N_GOCO06s in WGS84 lon/lat
  01b_mdt_ocean.png        — Raw MDT ocean values (NaN over land) in WGS84
  01b_datum_correction.png — Δ (MSL − EGM2008) applied to land pixels in UTM

Reference: Seeger & Minderhoud (2026), Nature. Method:
  - Geoid offset from ICGEM .gfc files via pyshtools spherical harmonic synthesis
  - MDT extrapolation over land: IDW (fillnodata, smoothing_iterations=1)
  - Only valid within 500 km of coastline (all deltas qualify)
  - DeltaDTM tiles already have this correction applied (suffix _GOCO06s_MDT)

5. RIVER NETWORK (rules 03d, 05, 06)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Rule 03d (get_river_network):
  - Clips SWORD global network to domain
  - Overlays DEM elevation: samples at ~100 m intervals, takes max finite value
  - Enforces MONOTONE DOWNSTREAM elevation decrease via Kahn's BFS topological
    sort: at each node, cap elev[dn] = min(elev[dn], elev[up])
  - Output: river_network.gpkg with max_elevation column

Rule 05 (clean_river_network):
  - BFS from boundary forcing crossing points downstream
  - Snaps crossings to nearest river reach (snapping_tolerance_m: 500)
  - Output: river_network_clean.gpkg

Rule 06 (river_depth):
  - Flow accumulation: iterative BFS from seed reaches (n_iterations: 100)
  - Bifurcation split: weight = width × angle_factor
    angle_factor: 180° (straight) → 1.0; 90° (perpendicular) → 0.1;
                  <90° (acute/U-turn) → 0.1; linear between 90–180°
    Angle computed from simplified reach geometries (simplify_tol=1e-3 degrees)
    upstream "back" vector: coords[-2] − coords[-1]
    downstream "forward" vector: coords[1] − coords[0]
  - Hydraulic depth from Leopold-Maddock: depth = alpha × Q^beta / width
    Params in config: a=0.27, b=0.30, c=7.20, f=0.50
    alpha = a×c, beta = b+f
  - Output: river_network_processed.gpkg with rivdph, bankfull_discharge_acc

Key functions in src/river_network.py:
  - build_downstream_adjacency()       — dict {reach_id: [dn_ids]}
  - collect_downstream_main_paths()    — BFS from seed set
  - accumulate_discharge()             — iterative BFS flow accumulation
    Width × angle_factor bifurcation split (angle factors pre-computed once)
  - compute_hydraulic_depth()          — Leopold-Maddock formula
  - _end_direction_vec(line, from_start, simplify_tol=1e-3)
  - _angle_factor(angle_deg, min_factor=0.1)

RIVER FORCING steps:
  Step 1:  Identify reaches that cross the domain grid boundary
  Step 2:  resolve_inside_domain_reaches() — walk downstream until reach

  Step 5:  Build forcing NetCDF

Upstream boundary check (07_test_upstream_boundary.py):
  Formula: wave_propagation_distance = T_eff × (celerity − v_river)
    T_eff    = effective_period_fraction × surge_period_hr × 3600 s
    celerity = sqrt(g × (depth_mouth + surge_amplitude))
    v_river  = max(min_river_velocity_ms, Q_bf / (W × D))
  Forcing points must be OUTSIDE the wave propagation radius from river mouths.
  Blocked mouths (v ≥ celerity): drawn as orange X markers (undetermined).
  Config params: effective_period_fraction (0.5), min_depth_at_mouth_m (2.0),
                 min_river_velocity_ms (1.0)
  Plot: geodesic circles (radius = wave propagation distance) + river network
        Green dot = forcing point outside circle (safe)
        Red dot   = forcing point inside circle (too close)

IDE FALSE POSITIVES:
  Snakefile: VS Code flags BASINS as undefined and expand() as unused.
  These are ALWAYS false positives — BASINS is defined in 00_common.smk via
  Snakemake include:, which the Python linter doesn't understand.
  GeoSeries.x / .y attribute errors in IDE are also false positives (pyright
  doesn't know the GeoSeries type).

RASTER ALIGNMENT:
  calculate_default_transform adds padding → extent > wgs84_bounds → visual
  misalignment. Fix: pass dst_bounds to _to_wgs84_grid() to use from_bounds.

MDT EXTRAPOLATION:
  MDT is NaN over land. Must call rasterio.fill.fillnodata() on the raw WGS84
  clip BEFORE resampling to UTM DEM grid. Load with 5° margin (mdt_load_margin_deg)
  to ensure sufficient offshore reference pixels.

SNAKEMAKE OUTPUT SYNC:
  Always keep _PREPROCESS_OUTPUTS in Snakefile in sync with rule outputs.
  MissingInputException for tracked outputs almost always means this list is stale.

SFINCS MODEL BUILD:
  HydroMT-SFINCS 2.0.0.dev0: use component-based API.
  Open SfincsModel in 'w+' mode for full build — NOT 'r+' (causes sfincs.ind errors).

RIVER ID HANDLING:
  reach_id from SWORD may be int or string; always normalise with
  str(int(float(x))) pattern before any dict/set operations.

BIFURCATION DISCHARGE:
  Weight = width × angle_factor. Single-arm transitions: angle correction
  irrelevant (full discharge passes through). Only applies at len(dn) > 1.

MONOTONE ELEVATION:
  Kahn's BFS (topological sort) in 03d enforces downstream elevation decrease.
  Cycles detected with a warning; DAG assumption is required for correctness.

OSM LAND POLYGONS:
  File has TWO layers: 'land_polygons' and 'marine_buffer'.
  gpd.read_file() defaults to the first layer ('marine_buffer') silently.
  Always specify layer="land_polygons" explicitly.

DEM ELEVATION FILTER:
  resolve_dem_elevation_reach() in src/river_forcing.py: walks downstream from
  inside-domain reach until max_elevation ≤ clip_elevation_m + buffer_m.
  NaN elevation treated as within range (unknown ≠ out of range).
